Genome update and mapping workflow

Author

Assa Yeroslaviz

Published

March 19, 2026

Overview

This project contains a conda-snakemake workflow for the mapping of the concatenated fastq file sequenced at the NGS facility of the MPIB.

The following documentation was meant for the sequencing center to help them run a preliminary analysis on the sequenced samples.

Following steps will be cover in this documentation

  1. Instruction on how to create the conda environment with the needed tools for the mapping and genomic indices updates
  2. The snakemake workflow for updating the genomic indices needed for the mapping
  3. Automation of themapping procedure using different mapping tools (Some tools are not yet incorporated in the workflow):
  • STAR
  • bowtie2
  • bwa
  • kallisto
  • salmon
  • cellranger

Analysis steps

This tool is straightforward to use.

First, one must first install the conda environment. Instruction for this step can be found in the environment file.

Updating the genomic indices should be done once, only when a new genome build is published. Ourwirkflow is using the Ensembl genomes fro various organisms. More can be added as needed. The list of genomes included in this project can be found in the genomes config file. At the top of this file one can also see the actulal version of the public repository.

We prepared a script called bcfngs. This script can take three global paraemters for each of the three steps assigned in this project

$ ./scripts/bcfngs --help
usage: bcfngs [-h] [--profile {micromamba,modules}] [--env-prefix ENV_PREFIX] {env,genomes,mapping} ...

positional arguments:
  {env,genomes,mapping}

options:
  -h, --help            show this help message and exit
  --profile {micromamba,modules}
  --env-prefix ENV_PREFIX
                        Micromamba environment prefix (default: cluster env if present, else .mamba/env).

The first parameter, bcfngs env take care of creating and maintaining the environent.

$ ./scripts/bcfngs env  --help
usage: bcfngs env [-h] {create,update,check} ...

positional arguments:
  {create,update,check}

options:
  -h, --help            show this help message and exit

Preparing the genomeic indicies

For the genomic indices we need the chromosome sequences as fastA file as well as the genomic features in a gtf format. The bcfngs genomes script can download the genomes list in the aforementioned file and create the indicies for the tool listed above.

A more elaborate instruction on how to run this step can be found in the Updating genomes files.

$ ./scripts/bcfngs genomes --help
usage: bcfngs genomes [-h] {check-updates,plan,build} ...

positional arguments:
  {check-updates,plan,build}

options:
  -h, --help            show this help message and exit

Mapping step

After we nesured, that our needed genomic indices are available for the tool we would like to use, we can as easily run the mapping step for the proper tool using the sam bcfngs mapping command

$ ./scripts/bcfngs mapping --help
usage: bcfngs mapping [-h] {plan,run} ...

positional arguments:
  {plan,run}

options:
  -h, --help  show this help message and exit

More elaborate instructions as to how to use this script for mapping the genomes with your favorite tool can be found in the mapping file.

Folder structure

To set the path were the genomes can be downloaded to use the genome config file. In our project we’re taking advantage of the pool-folders structire of our institute and store all genomic indices in the pool folder of the ngs core facility (pool-bcfngs).

genomes_root: /fs/pool/pool-bcfngs/genomes

Downloading the genomes creates a specific directory per organism, as can be seen below. The folder is split into the separate tools, also keeping the version of the tool that was used. To assertain reproducibility, we can keep previous version of genomics indices, using a softlink to point to the latest version of the indices.

The Workflow

The workflow uses a micromamba environment with the snakemake tool (based on python). The advantage of micromamba is having an isolated environment, separated from the rest of the tools installed on the system. This gives better control over the tool versions used in the workflow, with fewer conflicts from older system installations. To learn more about micromamba, take a look at the mamba documentation.

snakemake offers a lot of advantages as well. It allows better controls over different steps of the analysis, easier automation of the workflow and reduced running time through parallelization of the steps, just to mention some of them. to learn more about snakemake take a look at the snakemake doc site.

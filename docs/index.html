This website documents the BCF NGS mapping automation workflow. It is designed for reproducible, fast, and consistent preliminary analysis of sequencing projects at the core facility.

Why this tool exists

The same mapping tasks are repeated across many projects: environment setup, genome preparation, and standardized alignment/output generation. This tool provides one consistent interface so these steps are less error-prone, easier to review, and easier to run on local machines or HPC.

What the workflow can do

  • Build and maintain a dedicated micromamba environment for the pipeline.
  • Check whether a newer Ensembl release is available.
  • Download FASTA/GTF files and build versioned genome indices.
  • Build indices for multiple aligners (STAR, bwa, bowtie2).
  • Run preflight validation (lint, config checks, dry-run) before real execution.
  • Map single-end or paired-end FASTQ data to a selected organism.
  • Produce mapping outputs and normalized bigWig tracks for visualization.
  • Run with local execution or SLURM execution.

Main entry point: scripts/bcfngs

The main command-line wrapper is bcfngs (scripts/bcfngs). It is the recommended way to run the workflow end-to-end because it centralizes checks, planning, and execution.

Available command groups:

  • bcfngs env ... for environment management.
  • bcfngs genomes ... for genome download/index lifecycle.
  • bcfngs mapping ... for project mapping runs.

Available steps in bcfngs

  1. Environment setup and validation
    • bcfngs env create
    • bcfngs env update
    • bcfngs env check
  2. Genome maintenance
    • bcfngs genomes check-updates
    • bcfngs genomes plan
    • bcfngs genomes build
  3. Mapping workflow
    • bcfngs mapping plan (dry-run + checks)
    • bcfngs mapping run (real run)

Short usage summary

# 1) create/update and validate environment
./scripts/bcfngs env create
./scripts/bcfngs env check

# 2) (optional) check for newer Ensembl release
./scripts/bcfngs genomes check-updates

# 3) plan and build genomes/indices
./scripts/bcfngs genomes plan
./scripts/bcfngs genomes build

# 4) plan and run mapping
./scripts/bcfngs mapping plan --paired-end --mapper star
./scripts/bcfngs mapping run  --paired-end --mapper star

Configuration is split into:

  • configs/genomes.yaml for organisms, releases, and index settings.
  • configs/run.yaml for per-project mapping settings (project, input path, organism, output path).

11.12.2019

The workflow now should work on the HPC cluster of the MPI. For that reason some changes were made to the folder structure and the depth of the analysis.

  1. the mappers bowtie2 and bwa were removed from the analysis. It will be done from now on only with STAR.
  2. the working directory of the analysis should be the conc.fastq folder in the pool-bcfngs pool folder of the Sequencing core facility.
  3. The workflow will create a new folder with the structure ProjectNumber/Organism/ where it will save two new folders for the star mapping results and the bigwig files.
  4. the first script to get the genomes should be only run once, or when the genome version must be updated.
  5. The second script was split into single-end and paired-end versions and should be run accordingly.

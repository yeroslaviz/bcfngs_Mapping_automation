Updating genomes and indices

Author

AY

This page collects the internal procedures for:

  • updating genome FASTA/GTF sources
  • rebuilding indices (STAR, bwa, bowtie2)
  • switching the active index used by mapping (symlinks)
  • running mapping for single-end / paired-end projects

When to update genomes / indices

Once every few years the genomic sequences must be updated. Also the STAR version for mapping progresses and should be updated when needed.

Prerequisites / where to run

You can use any server as long as you can run snakemake and the required tools. The recommended method is micromamba (see environment). Default environment path: ~/miniconda3/envs/update_genomes

On the HPC cluster this expands to: /fs/gpfs41/lv07/fileset07/home/b_cox/yeroslaviz/miniconda3/envs/update_genomes

How-to (by scenario)

First time setup (recommended)

./scripts/bcfngs env create
./scripts/bcfngs env check

Routine: re-run indexing (no config changes)

./scripts/bcfngs genomes plan
./scripts/bcfngs genomes build --cores 20

Only rebuild one or two organisms

./scripts/bcfngs genomes build --cores 20 --organism Homo_sapiens,Mus_musculus

Ensembl published a new release

  1. Check:
./scripts/bcfngs genomes check-updates
  1. If an update is available, bump releases.ensembl in configs/genomes.yaml.
  2. Then:
./scripts/bcfngs genomes plan
./scripts/bcfngs genomes build --cores 20

You updated STAR/bwa/bowtie2 (rebuild indices into new versioned folders)

  1. Update tools:
./scripts/bcfngs env update
./scripts/bcfngs env check
  1. Update index_versions in configs/genomes.yaml (these strings are used in output folder names).
  2. Rebuild:
./scripts/bcfngs genomes build --cores 20

HPC modules fallback (aligners from modules)

module load bclconvert/4.3.13 star bowtie2 bwa
./scripts/bcfngs --profile modules env check
./scripts/bcfngs --profile modules genomes plan
./scripts/bcfngs --profile modules genomes build --cores 20

Recommended (micromamba)

./scripts/bcfngs env create
./scripts/bcfngs env check

HPC modules fallback (optional)

You can use modules for vendor tools:

module load bclconvert/4.3.13 star bowtie2 bwa

1) Check for a new Ensembl release (report only)

./scripts/bcfngs genomes check-updates

2) Update genome sources (if needed)

Genome sources are configured in configs/genomes.yaml. Organism identifiers use scientific names (e.g. Homo_sapiens, Mus_musculus).

To add or update a genome:

  1. Add/update the organism entry under organisms:.
  2. For Ensembl sources, keep releases.ensembl pinned (default: 116). Bumping the release is a controlled change.
  3. Set sa_index appropriately for STAR indexing.

3) Build genomes + indices (safe workflow)

Run preflight checks first:

./scripts/bcfngs genomes plan

Then build:

./scripts/bcfngs genomes build --cores 20

To build only a subset:

./scripts/bcfngs genomes build --cores 20 --organism Homo_sapiens,Mus_musculus

4) Versioned indices + symlinks

Indices are built into versioned directories (e.g. starIndex-<version>) and then symlinked so mapping can always use stable paths:

  • /fs/pool/pool-bcfngs/genomes/<Organism>/starIndex
  • /fs/pool/pool-bcfngs/genomes/<Organism>/bwaIndex
  • /fs/pool/pool-bcfngs/genomes/<Organism>/bowtie2Index

Symlinks are updated automatically by the Snakemake workflow after successful index creation.

5) Mapping (per project)

Genome download and indexing are handled by ./scripts/bcfngs. Mapping is still run through the STAR Snakefiles already in the repository.

Change parameters before mapping

Adjust configs/run.yaml:

project: "P152"
path: "/fs/pool/pool-bcfngs/fastq_files/P152/conc.fastq/"
outdir: "/fs/pool/pool-bcfngs/mapping"
organism: "Drosophila_melanogaster"
threads: 16

Run mapping (dry-run first)

cd /fs/pool/pool-bcfngs/fastq_files/P152/conc.fastq/

# paired-end
snakemake -nps /path/to/repo/Star.MappingQuant.PairedEndFastq.Snakefile -j 20

# single-end
snakemake -nps /path/to/repo/Star.MappingQuant.SingleEndFastq.Snakefile -j 20

Remove -n to run the mapping for real after the dry-run looks correct.

Environment (micromamba + bcfngs)

This page focuses on the workflow environment itself: what it contains, how bcfngs env manages it, and practical command examples.

The environment is defined in env/environment.yml and is managed through:

  • ./scripts/bcfngs env create
  • ./scripts/bcfngs env update
  • ./scripts/bcfngs env check

What bcfngs env controls

bcfngs env wraps micromamba and standardizes environment setup across users and servers.

It does three things:

  • create: creates a new environment from env/environment.yml
  • update: updates an existing environment from env/environment.yml
  • check: validates required tools and prints detected versions as JSON

Command syntax and options

./scripts/bcfngs [--profile micromamba|modules] [--env-prefix PATH] env <create|update|check>

--profile

  • micromamba (default): run tools from the micromamba environment
  • modules: load HPC modules first, then run checks in the selected env

Note: env create and env update require --profile micromamba.
--profile modules is intended for env check.

--env-prefix

Environment path prefix. If omitted, bcfngs uses its built-in default path.

Common choices:

  • project-local: .mamba/env
  • shared cluster path: ~/miniconda3/envs/update_genomes

Installing micromamba

Install micromamba from the micromamba releases.

"${SHELL}" <(curl -L https://micro.mamba.pm/install.sh)

Examples for all bcfngs env options

1) env create

Create at the default environment prefix:

./scripts/bcfngs env create

Create at a custom path:

./scripts/bcfngs --env-prefix ~/miniconda3/envs/update_genomes env create

Create explicitly with micromamba profile (equivalent to default profile):

./scripts/bcfngs --profile micromamba --env-prefix ~/miniconda3/envs/update_genomes env create

2) env update

Update environment at the default prefix:

./scripts/bcfngs env update

Update a custom environment path:

./scripts/bcfngs --env-prefix ~/miniconda3/envs/update_genomes env update

Update explicitly with micromamba profile:

./scripts/bcfngs --profile micromamba --env-prefix ~/miniconda3/envs/update_genomes env update

3) env check

Check default environment:

./scripts/bcfngs env check

Check a custom environment:

./scripts/bcfngs --env-prefix ~/miniconda3/envs/update_genomes env check

Check with HPC modules profile:

./scripts/bcfngs --profile modules --env-prefix ~/miniconda3/envs/update_genomes env check

Example output:

{
  "snakemake": "8.30.0",
  "STAR": "2.7.11b",
  "bwa": "...",
  "bowtie2": "...",
  "samtools": "samtools 1.23"
}

Common workflows

First-time setup (recommended)

./scripts/bcfngs --env-prefix ~/miniconda3/envs/update_genomes env create
./scripts/bcfngs --env-prefix ~/miniconda3/envs/update_genomes env check

Routine update

./scripts/bcfngs --env-prefix ~/miniconda3/envs/update_genomes env update
./scripts/bcfngs --env-prefix ~/miniconda3/envs/update_genomes env check

Run a one-off command without activating

micromamba run -p ~/miniconda3/envs/update_genomes snakemake --help
micromamba run -p ~/miniconda3/envs/update_genomes STAR --version

Notes and limitations

  • --profile modules does not support env create or env update.
  • env check verifies the core toolchain (python, snakemake, STAR, bwa, bowtie2-build, samtools, curl).
  • If you need reproducibility snapshots, export explicit package lists after updates:
mkdir -p env/locks
micromamba list -p ~/miniconda3/envs/update_genomes --explicit > env/locks/explicit-YYYY-MM-DD.txt

Environment (micromamba)

Author

AY

Published

March 19, 2026

This workflow uses micromamba to create a reproducible environment with all required tools.

Default environment location

The commands in this repository assume this default path:

~/miniconda3/envs/update_genomes

On the HPC cluster this expands to:

/fs/gpfs41/lv07/fileset07/home/b_cox/yeroslaviz/miniconda3/envs/update_genomes

The wrapper ./scripts/bcfngs uses the same path by default. If you keep the environment somewhere else, add --env-prefix /path/to/env.

Installing micromamba

micromamba can be installed from the github repository.

"${SHELL}" <(curl -L https://micro.mamba.pm/install.sh)

Create the workflow environment

From the repo root:

micromamba create -y -p ~/miniconda3/envs/update_genomes -f env/environment.yml

If your micromamba root prefix is already ~/miniconda3, the equivalent named-environment command is:

micromamba create -y -n update_genomes -f env/environment.yml

Run commands without activating the environment

micromamba run -p ~/miniconda3/envs/update_genomes snakemake --version
micromamba run -p ~/miniconda3/envs/update_genomes STAR --version

Update the environment

If env/environment.yml changes:

micromamba update -y -p ~/miniconda3/envs/update_genomes -f env/environment.yml
./scripts/bcfngs env check

Health checks

./scripts/bcfngs env check

Example output:

{
  "snakemake": "8.30.0",
  "STAR": "2.7.11b",
  "bwa": "",
  "bowtie2": "/fs/gpfs41/lv07/fileset07/home/b_cox/yeroslaviz/miniconda3/envs/update_genomes/bin/bowtie2-build-s version 2.5.5",
  "samtools": "samtools 1.23"
}

Troubleshooting missing tools

If ./scripts/bcfngs env check reports a missing tool:

  1. Update the environment:
./scripts/bcfngs env update
./scripts/bcfngs env check
  1. If you use a non-default location, pass it explicitly:
./scripts/bcfngs --env-prefix /path/to/your/env env update
./scripts/bcfngs --env-prefix /path/to/your/env env check
  1. Verify the package is installed in the expected environment:
micromamba list -p ~/miniconda3/envs/update_genomes | grep -Ei 'snakemake|star|bwa|bowtie2|samtools|rseqc'
micromamba run -p ~/miniconda3/envs/update_genomes snakemake --version
  1. If needed, recreate the environment:
rm -rf ~/miniconda3/envs/update_genomes
./scripts/bcfngs env create
./scripts/bcfngs env check

How-to (common scenarios)

First-time setup

./scripts/bcfngs env create
./scripts/bcfngs env check

Run a one-off command without activating

micromamba run -p ~/miniconda3/envs/update_genomes snakemake --help
micromamba run -p ~/miniconda3/envs/update_genomes STAR --version

Keep tools up to date

./scripts/bcfngs env update
./scripts/bcfngs env check

Optional lockfile:

mkdir -p env/locks
micromamba list -p ~/miniconda3/envs/update_genomes --explicit > env/locks/explicit-YYYY-MM-DD.txt

Use HPC modules for vendor tools

You can keep python and snakemake in micromamba while loading aligners from modules:

module load bclconvert/4.3.13 star bowtie2 bwa
./scripts/bcfngs --profile modules env check
./scripts/bcfngs --profile modules genomes plan
./scripts/bcfngs --profile modules genomes build --cores 20

Demultiplexing with bcl-convert

This repo does not automate demultiplexing yet. Typical usage on the cluster is:

module load bclconvert/4.3.13
bcl-convert --help

Once every few years the genomic sequences must be updated. Aslo the STAR version for the mapping is progressing and must be updated, if needed.

The way to go is as follow:

1. update the fastA and gtf files

The config file used for the genome indexing and mapping is "config.yaml" (in the pool-bcfngs folder). This file contains a list of links of a fastA and a gtf file for each genome.
Every once in a while this links should be updated. I'm not sure how often it make sense to do so, but at least when the STAR version is updated, it also make sense to bring this to the newest genome version.

The links all point to the current_fasta or current_gtf folders of each of the organisms in the Ensembl repository. If the version was updarted, it also good to update the name-prefix of the organism, which is also the name of the folder for the mapping.

An example:
the older D. mel version
Dme.BDGP6.22:
  fasta: "ftp://ftp.ensembl.org/pub/current_fasta/drosophila_melanogaster/dna/Drosophila_melanogaster.BDGP6.22.dna.toplevel.fa.gz"
  gtf: "ftp://ftp.ensembl.org/pub/current_gtf/drosophila_melanogaster/Drosophila_melanogaster.BDGP6.22.91.gtf.gz"
  SAindex: 12

was renewed to this one:
Dme.BDGP6.28:
  fasta: "ftp://ftp.ensembl.org/pub/current_fasta/drosophila_melanogaster/dna/Drosophila_melanogaster.BDGP6.28.dna.toplevel.fa.gz"
  gtf: "ftp://ftp.ensembl.org/pub/current_gtf/drosophila_melanogaster/Drosophila_melanogaster.BDGP6.28.102.gtf.gz"
  SAindex: 12

2. rerun the indexing process

The snakemake file "get_STARIndex.Snakefile" contains the rules used for the STAR mapping.

The only thing one must changes here is the STAR outut folder with the current version. This must be done in two places. One in rule all in row 14 and one in rule star_index in row 40.

An example:
change This -
    expand("{org}/starIndex2.7.3a/", org=config['organism']),

to that -
    expand("{org}/starIndex2.7.7a/", org=config['organism']),
    

```
snakemake -n -j 20 -s ../scripts/getGenome_IndexGenome.Snakefile  
```

The -n parameter ensure a dry-run before we do the actual update. 
-j accounts for the number of cores to be used. 


3. create a symlink to the new genomic folder for the STAR index.

The DoMappping.pl script in the ngs-vm runs the mapping. the script can be found under - /www/mpiseq2/bin/doMapping.pl. In general, it points to the index folder starIndex/ under the path - /fs/pool/pool-bcfngs/genomes/{organism}/starIndex/.

So the best way to keep the paths correct would be to create a folder for each STAR version and create a symlink to the folder starIndex.
This can be done as such.
first make sure the the old starIndex is deleted. Then type the following command:


folders="Cel.WBcel235
Dme.BDGP6.28
Dps.3.0.49
Dre.GRCz11
Eco.HUSEC2011CHR1
Hsp.GRCh38
Mmu.GrCm38
Sce.R64-1-1"

for folder in $folders
do 
  echo $folder
  cd ${folder}/
  rm starIndex
  ln -s  starIndex2.7.10a/ starIndex
  cd ../
done

The loop will go through all the folders and creates a soft link from the new genome index folder to the folder starIndex, including all the files within.

4. When this is finished the mapping can be done using a second snakemake script, the `Star.MappingQuant.SingleEndFastq.Snakefile` qhich uses the same config file to get some of the parameters for the run, such as the project number and path to the fastq files. 

######
###     HOW TO WORK THIS script
###  1. log into any of the servers or hpcl
###  2. start a screen session (e.g. JacqueB)
###  3. activate the conda env "JacqueB"
###  4. change to the folder of the fastq files (e.g./fs/pool/pool-bcfngs/fastq_files/P468/P468_20211202_JM_JB/conc.fastq).
###  5. Modify the two script (Snakemake and config) accordingly:
###     5.1 rename to the file based on the project number (e.g. P486, ...)
###     5.2 In the Snakefile - change the path to the config file (e.g. configfile:"/fs/pool/pool-bcfngs/scripts/P468.config.yaml" )
###     5.3 In the config file - in row 31 change the project name as well (e.g. "P486")
###     5.4 In the config file - in row 35 change the path to the conc.fastq fiolder (should be identical to the one mae in step 4)
###     5.5 In the config file - in row 39 choose what organism you would like to map against.
###  6. copy both files to the pool-bcfngs/scripts folder.
###  7. in the terminal change to the folder of fastq file (see step 4)
###  8. run the snakemake command
###     8.1 first as a dry run - snakemake -nps /fs/pool/pool-bcfngs/scripts/P468.Star.MappingQuant.PairedEndFastq.Snakefile -j 16
###     8.2 if all goes well, then run it again without the -n parameter.
#######

```
snakemake -nps /fs/pool/pool-bcfngs/scripts/Star.MappingQuant.PairedEndFastq.Snakefile -j 10
```

or if we have a single-end run:

```
snakemake -nps /fs/pool/pool-bcfngs/scripts/Star.MappingQuant.SingleEndFastq.Snakefile -j 10
```

-n ensure a dry-run to check that everything is correct. to actually do the analysis, the `-n` parameter must be removed.

Branch and Publish Workflow (Local Notes)

Author

AY

Published

March 18, 2026

1 Strict branch split

  • main: source branch for code, config, and docs source.
  • gh-pages: publish-only branch managed by quarto publish gh-pages.

2 Allowed files by branch

  • main examples:

  • scripts/

  • workflow/

  • configs/

  • env/

  • *.qmd

  • _quarto.yml

  • Snakefiles and helper code

  • gh-pages examples:

  • Generated site files only.

  • No hand edits.

3 New publish route

git switch main
# edit source files: code, config, docs source
git add <source_files>
git commit -m "feat/docs: ..."
git push origin main
quarto render
quarto publish gh-pages

Use _site/, not _sites/. _site/ is Quarto’s standard local build directory and is already ignored by git.

4 What changes with this setup

  • No more manual git switch gh-pages for publishing.
  • No more git add docs/ from the source branch.
  • No more temporary checkout of resource files from another branch before rendering.
  • gh-pages becomes a generated-output branch, not a working branch.

5 Why resources stay current automatically

When project.resources includes files like scripts/bcfngs, env/*, configs/*, and workflow/*, Quarto copies whatever exists on the current branch during quarto render.

With the new route, render and publish happen from main, so the copied resources are always the newest versions from that source branch.

git switch main
# make source changes
git add scripts/bcfngs env/environment.yml configs/run.yaml workflow/mapping.smk *.qmd _quarto.yml
git commit -m "feat/docs: update workflow and docs"
git push origin main
quarto render
quarto publish gh-pages

6 When to publish

  • Publish when docs source changes.
  • Publish when files listed in project.resources change and you want the website downloads/copies refreshed.
  • Skip publish for code-only changes that do not affect the site.

7 Preview locally

git switch main
quarto render

Preview output will be written to _site/.

8 One-time cleanup note

  • The current docs/ directory is a legacy output path from the manual workflow.
  • After switching to quarto publish gh-pages, do not use docs/ as the local render target anymore.
  • Old tracked docs/ files on existing branches can be cleaned up separately once the new publish route is working.

9 Non-negotiable rule

  • Do not hand-edit gh-pages.
  • Do not commit _site/ to codex/hpc-test.
  • Keep source changes on main, then publish with Quarto.

Mapping Workflow

Author

AY

Published

March 19, 2026

This page documents the current mapping workflow in this repository.

Overview

  • Genome download and indexing are handled by ./scripts/bcfngs genomes ....
  • Mapping is now implemented in workflow/mapping.smk.
  • The recommended entry point is ./scripts/bcfngs, using mapping plan for validation and dry-run and mapping run for execution.
  • The workflow supports star, bwa, and bowtie2.
  • For every selected mapper, the workflow creates BAM, BAM index, and bigWig output files.

If star is selected, the workflow also writes:

  • <sample>.counts.tab
  • <sample>.SJ.out.tab

Required configuration

Base project settings are stored in configs/run.yaml:

project: "P000"
path: "/fs/pool/pool-bcfngs/fastq_files/P000/P000_Testrun/conc.fastq/"
outdir: "/fs/pool/pool-bcfngs/mapping"
organism: "Caenorhabditis_elegans"
threads: 16
ram_bytes: 168632718037

Important fields:

  • project: used in the output layout
  • path: FASTQ input directory
  • outdir: output root directory
  • organism: one organism key from configs/genomes.yaml
  • threads: mapping threads per job
  • ram_bytes: STAR BAM sort memory limit

The wrapper also allows runtime overrides such as:

  • --project
  • --organism
  • --fastq-dir
  • --outdir
  • --threads
  • --mapper
  • --paired-end or --single-end

Required genome resources

Before mapping starts, the workflow checks that the needed indices and chromosome size file already exist under the configured genomes root.

Depending on the mapper selection, it expects:

  • star: <genomes_root>/<organism>/starIndex/SA
  • bwa: <genomes_root>/<organism>/bwaIndex/<organism>.bwt
  • bowtie2: <genomes_root>/<organism>/bowtie2Index/<organism>.1.bt2
  • all runs: <genomes_root>/<organism>.chromSize

If one of these files is missing, the run stops before any mapping jobs are submitted.

FASTQ pattern auto-detection

The workflow detects the input pattern directly from the FASTQ directory. The supported R1 patterns are:

  1. *_R1.fastq.gz
  2. *_R1_001.fastq.gz
  3. *_L1_R1.fastq.gz
  4. *.conc.R1.fastq.gz

For paired-end mode, the matching R2 files must also exist.

Output layout

Outputs are written below:

<outdir>/<project>/<organism>/

For each selected mapper, the workflow creates:

<outdir>/<project>/<organism>/star/<sample>.bam
<outdir>/<project>/<organism>/star/<sample>.bam.bai
<outdir>/<project>/<organism>/star/bigwig/<sample>.bw
<outdir>/<project>/<organism>/star/<sample>.counts.tab
<outdir>/<project>/<organism>/star/<sample>.SJ.out.tab

<outdir>/<project>/<organism>/bwa/<sample>.bam
<outdir>/<project>/<organism>/bwa/<sample>.bam.bai
<outdir>/<project>/<organism>/bwa/bigwig/<sample>.bw

<outdir>/<project>/<organism>/bowtie2/<sample>.bam
<outdir>/<project>/<organism>/bowtie2/<sample>.bam.bai
<outdir>/<project>/<organism>/bowtie2/bigwig/<sample>.bw

<outdir>/<project>/<organism>/logs/

Basic usage

Dry-run and validation

Use plan first. This validates config and inputs, runs Snakemake lint, and performs a dry-run.

./scripts/bcfngs mapping plan \
  --mapper star \
  --paired-end \
  --project P000 \
  --organism Caenorhabditis_elegans \
  --fastq-dir /fs/pool/pool-bcfngs/fastq_files/P000/P000_Testrun/conc.fastq \
  --outdir /fs/pool/pool-bcfngs/mapping

Local run

./scripts/bcfngs mapping run \
  --mapper star \
  --paired-end \
  --project P000 \
  --organism Caenorhabditis_elegans \
  --fastq-dir /fs/pool/pool-bcfngs/fastq_files/P000/P000_Testrun/conc.fastq \
  --outdir /fs/pool/pool-bcfngs/mapping \
  --executor local \
  --jobs 20

SLURM run

./scripts/bcfngs mapping run \
  --mapper star,bwa \
  --paired-end \
  --project P000 \
  --organism Caenorhabditis_elegans \
  --fastq-dir /fs/pool/pool-bcfngs/fastq_files/P000/P000_Testrun/conc.fastq \
  --outdir /fs/pool/pool-bcfngs/mapping \
  --executor slurm \
  --jobs 20 \
  --slurm-partition p.hpcl93 \
  --slurm-time 04:00:00 \
  --slurm-cpus-per-task 20

Direct Snakemake usage

The workflow can also be called directly:

snakemake \
  --printshellcmds \
  --dry-run \
  -s workflow/mapping.smk \
  --configfile configs/run.yaml \
  --config \
    project=P000 \
    organism=Caenorhabditis_elegans \
    fastq_dir=/fs/pool/pool-bcfngs/fastq_files/P000/P000_Testrun/conc.fastq \
    outdir=/fs/pool/pool-bcfngs/mapping \
    mappers=star,bwa \
    read_mode=paired \
    threads=16 \
    ram_bytes=168632718037 \
    bw_bin_size=10 \
    bw_normalization=CPM

Notes

  • The workflow supports exactly one organism per run.
  • Unknown mapper names are rejected.
  • Duplicate mapper names are removed automatically.
  • The default read mode is paired-end. Use --single-end explicitly for single-end data.
  • STAR intermediary files are removed unless --keep-intermediary-files is set.
  • bigWig files are produced directly with bamCoverage. The old wig conversion step is no longer part of this workflow.
  • Run ./scripts/bcfngs env check first if you want to verify that snakemake, STAR, bwa, bowtie2-build, samtools, and bamCoverage are available in the configured environment.

Useful flags

  • --mapper: comma-separated mapper list such as star,bwa,bowtie2
  • --threads: override mapping threads from configs/run.yaml
  • --ram-bytes: override STAR --limitBAMsortRAM
  • --bw-bin-size: set bamCoverage --binSize
  • --bw-normalization: set bamCoverage --normalizeUsing
  • --jobs: maximum concurrent jobs
  • --executor: choose local or slurm
